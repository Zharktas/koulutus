Web Applications

  • The Web (World Wide Web, www)

    is an information system of interlinked hypertext documents accessed via the Internet, using web browser

  • A web application

    is any software that runs in a web browser

History of Web Applications

  • 1990's

    • Mostly static HTML pages
    • Eg. http://www.cs.tut.fi/~prog2/
    • "Web 1.0"

  • 2000's

    • Dynamic web applications
    • User-generated content
    • Eg. http://www.wikipedia.org
    • "Web 2.0"

  • 2010's

    • HTML5, user experience comparable to desktop applications
    • Eg. http://play.spotify.com
    • "Web 3.0"

Why Web Applications?

  • Pros

    • Available everywhere
      • Platform-independent
      • The only requirement is a modern web browser
    • No installation
    • Automatic updates
    • Safer to run than native application (browser sandbox)

  • Cons

    • Requires an internet connection
    • Own kind of security concerns
      • The application is reachable by the whole world
    • Web technology limitations
      • Not that big issue nowadays

Web Application Architecture

  • Web application consist of

    1. front end (run on a web browser)
    2. back end (run on a web server)

  • The communication between the two is done using Hypertext Transfer Protocol (HTTP)

  • The back end of a web application can be implemented using various technologies, programming languages, and frameworks

Front-end technologies

  • HTML

    • Describes the content and structure of a web page

  • CSS

    • Describes the appearance of a web page

  • JavaScript

    • The programming language of a web browser
    • Implements the functionality on the web page

HTML

  • HyperText Markup Language

  • Describes the structure of a web page

    • A tree of HTML elements
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Hello World</title>
  </head>
  <body>
    <h1 class="title">Hello</h1>
    <p id="first">First!</p>
    <p>Second!</p>
  </body>
</html>

CSS

  • Cascading Style Sheets

  • Describes the appearance of a web page

  • Selectors with attribute-value pairs

  • hellostyle.css 
/* Select by element name */
p {
  background-color: orange;
}

/* Select by class */
.title {
  color: red;
}

/* Select by id */
#first {
  border: 1px solid black;
  padding: 10px;
}

<!DOCTYPE html>
<html lang="en">

  </head>

JavaScript

  • The programming language of a web browser

  • Dynamically typed

  • First-class functions

  • Prototype-oriented

  • Can be attached on a HTML page using the <script> tag

    <script>
  alert("Hello");
</script>
    or 
    <script src="file.js"></script>

JavaScript Syntax

  • JavaScript syntax is somewhat similar to C++
var x = 100;
var y = "Hello";
var li = [1, 2, 3];
for (var i=0; i<100; i++) {
  // ...
}
while (something()) {
  ...
}
function plus(a, b) {
  return a + b;
}

Dynamic typing

  • JavaScript is a dynamically typed language

  • Variables have no type, values do

    • Everything is a var

  • Same variable can contain numbers, strings, objects, lists, functions, booleans, ...

var x = "Hello";
x = 100;
x = true;
x = [1, 2, "Three"];

JavaScript Objects

  • JavaScript has objects, but no classes
var obj1 = {
  name: "Jack",
  age: 20
};
  • JavaScript is very flexible
    • Objects can be arbitrarily modified during their lifetime
obj1.name = "Jill";
obj1.height = 160;
delete obj1.age;
  • Instead of classes, JavaScript has a concept called prototypes

First-class Functions

  • JavaScript has first-class functions

  • Functions are like any other values

    • they can be assigned to variables, passed as parameters, returned from functions, ...
function twice(func)  {
  return function() {
    func();
    func();
  }
}

function sayHi() {
  alert("Hi!");
}

var sayHiTwice = twice(sayHi);

sayHiTwice();

  • script.js
"use strict";

function pClicked(event) {
  event.srcElement.style.
    backgroundColor = 'red';
}

var ps = document.
  getElementsByTagName('p');

for(var i=0; i<ps.length; ++i){
  var p = ps[i];
  p.onclick = pClicked;
}

Ajax

  • Ajax is a set of technologies that allows a web page to communicate with a server without reloading the whole page

  • "Asynchronous JavaScript And XML"

  • Web browser provides the XMLHttpRequest object

    • Allows making arbitrary HTTP requests
var xhr = new XMLHttpRequest();
  xhr.open('GET', '/url/to/backend');
  xhr.onload = function(e) {
    alert(this.response);
  }
  xhr.send();
