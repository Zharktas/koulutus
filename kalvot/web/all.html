Kalvot

World Wide Web

  • World Wide Web (WWW, Web) on web-resurssien muodostama verkko

  • Web-resurssit voivat olla HTML-sivuja, JavaScript-ohjelmia, datatiedostoja, kuvia, videoita tai lähes mitä tahansa muuta

  • Kaikilla web-resursseilla on URL (Uniform Resource Locator), jonka avulla resurssiin voidaan viitata

Web vs Internet

  • Internet on maailmanlaajuinen "verkkojen verkkojen", jossa dataa siirretään TCP/IP-protokollalla

  • Internetissä voidaan siirtää monenlaista dataa, kuten sähköposteja, verkkopeliliikennettä ja web-resursseja

  • Web-resurssien siirtoon käytetty protokolla on nimeltään HTTP (ja HTTPS ja FTP), joka puolestaan hyödyntää TCP/IP:tä

  • Web siis "toimii Internetin päällä"

Webin historiaa

  • 1969: ARPANET

    • Internetin edeltäjä

  • 1983: TCP/IP

    • Internetin tiedonsiirtoprotokolla

  • 1989: World Wide Web

    • HTTP, HTML

  • 1995: JavaScript

    • Web-selainten ohjelmointikieli

  • 1996: CSS

    • Kieli web-sivujen tyylin määrittelyyn

  • 1998: Ajax

    • Monipuolisempi selain-palvelin-kommunikaatio

  • 2011: HTML5

    • Joukko erilaisia web-teknologioita

Web-sovellusten historiaa

  • 1990-luku

    • Suurelta osin staattisia HTML-sivuja
    • Esim. http://www.cs.tut.fi/~jkorpela/
    • "Web 1.0"

  • 2000-luku

    • Dynaamisia web-sovelluksia
    • Käyttäjien luomaa sisältöä
    • Esim. http://www.wikipedia.org
    • "Web 2.0"

  • 2010-luku

    • HTML5, työpöytäsovelluksen tasoinen käyttäjäkokemus
    • Semanttinen web, Mashupit, ...
    • Esim. http://play.spotify.com
    • "Web 3.0"

Web-sovellus

  • en.wikipedia.org:

    Web application is a client-server software application in which the client (or user interface) runs in a web browser.

Web-sivu vs web-sovellus

  • Web-sivu: käyttäjä hakee tietoa

    • esim. http://www.tut.fi/edutech

  • Web-sovellus: käyttäjä suorittaa toimintoja

    • esim. https://gmail.com

  • Rajanveto epätarkkaa

  • Keskustelua aiheesta

Web-sovellusten hyötyjä

(verrattuna natiiviin työpöytä-/mobiilisovellukseen)

  • Käytettävissä kaikkialla

    • Alustariippumaton
    • Ainut vaatimus on moderni web-selain

  • Sovellusten päivittäminen helppoa

  • Turvallisempi suorittaa kuin natiivi sovellus

    • Web-selaimen "hiekkalaatikko"

Web-sovellusten haittoja

  • Käyttäjäkokemus?

  • Käyttö vaatii verkkoyhteyden

  • Omanlaisensa tietoturvaongelmat

    • Sovellus koko maailman saavutettavissa

  • Sovellusten kehitys

    • Teknologiarajoitteet
    • Web-selainten erot ym.

Kaikki nämä ongelmat ovat vähenemään päin web-teknologioiden kehittymisen myötä.

Web-teknologioita

  • HTML

    • määrittelee web-sivun sisällön

  • CSS

    • kuvaa web-sivun ulkoasun

  • JavaScript

    • web-sivun toiminnallisuus

  • HTTP

    • kommunikointi web-selaimen ja palvelimen välillä

Tyypillinen web-sovellus

  1. Selainpää (Frontend), suoritetaan web-selaimessa
    • HTML, CSS, JavaScript

  2. Palvelinpää (Backend)

    • Lukuisia vaihtoehtoisia toteutustapoja, sovelluskehyksiä, ohjelmointikieliä, ...
    • Yhteys tietokantaan

  3. Näiden välillä kommunikoidaan HTTP-protokollalla

HTTP

  • HyperText Transfer Protocol

  • Protokolla asiakkaan (usein web-selain) ja palvelimen väliseen kommunikaatioon

  • Tekstipohjainen

  • Tilaton

HTML

  • HyperText Markup Language

  • Merkkauskieli web-sivun sisällön kuvaamiseen

Web-selaimen kehitystyökalut

  • Web-selaimissa on kehitystyökaluja, joilla voi tarkkailla HTTP-liikennettä, sivustojen lähdekoodeja (HTML, CSS, JavaScript, ym.)

  • Firefoxissa kehitystyökalut saa auki pikanäppäimellä Ctrl+Shift+C

Harjoitus

Avaa web-sivu osoitteessa http://www.w3.org ja yritä selaimen kehitystyökalujen (erityisesti Network-välilehti) avulla löytää vastaukset seuraaviin kysymyksiin:

  1. Mitä resursseja selain lataa kun sivuston avaa?

  2. Minkä tyyppisiä resursseja?

  3. Mikä resurssi ladataan ensimmäisenä?

  4. Mistä selain tietää mitä resursseja pitää ladata sen jälkeen kun alkuperäinen http://www.w3.org -resurssi on ladattu?
